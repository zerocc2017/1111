SDWebImage 源码解析一

发表于 2016-02-28
字数统计:

SDWebImage 源码阅读，读其技巧，学其思想。SDWebImage 框架其提供了图片从下载、解析、处理、缓存、清理等一些列功能，是一个异步下载图片并支持缓存的 UIImageView 分类，使用简洁灵活。

框架中所有类浅析

一、各分类(提供直接使用的接口)

  • UIImageView(WebCache) 为 UIImageView 分类，扩展了加载图片的方法；
  • UIImageView(HighlightedWebCache) 为 UIImageView 分类，扩展了高亮状态下加载图片的方法；
  • UIButton(WebCache) 为 UIButton 分类，扩展了加载载图片的方法；
  • MKAnnotationView(WebCache) 为 MKAnnotationView 分类，扩展了其各种加载图片的方法；
  • UIView(WebCache)

二、工具类

  • NSData+ImageContentType 根据图片数据获取图片的类型，比如GIF、PNG等;
  • UIImage+GIF 对于一张图片是否GIF做判断。可以根据 NSData 返回一张 GIF 的UIImage 对象，并且只返回 GIF 的第一张图片生成的 GIF;
  • UIImage+MultiFormat 获取 UIImage 对象对应的 data、或者根据 data 生成指定格式的UIImage，其实就是 UIImage 和 NSData 之间的转换处理;
  • SDWebImageDecoder 根据图片的情况，做图片的解压缩处理。并且根据图片的情况决定如何处理解压缩;

三、核心类

  • SDWebImageManager 拥有一个 SDWebImageCache 和 SDWebImageDownloader 属性分别用于图片的缓存和加载处理。为UIView及其子类提供了加载图片的统一接口。管理正在加载操作的集合。这个类是一个单列。还有就是各种加载选项的处理。
  • SDImageCache 负责 SDWebImage 的整个缓存工作，是一个单列对象。缓存路径处理、缓存名字处理、管理内存缓存和磁盘缓存的创建和删除、根据指定key获取图片、存入图片的类型处理、根据缓存的创建和修改日期删除缓存。
  • SDWebImageDownloader 实现了图片加载的具体处理，如果图片在缓存存在则从缓存区。如果缓存不存在，则直接创建一个。SDWebImageDownloaderOperation 对象来下载图片。管理 NSURLRequest 对象请求头的封装、缓存、cookie 的设置。加载选项的处理等功能。管理 Operation 之间的依赖关系。这个类是一个单列.
  • SDWebImageDownloaderOperation 一个自定义的并行 Operation 子类。这个类主要实现了图片下载的具体操作、以及图片下载完成以后的图片解压缩、Operation 生命周期管理等。
